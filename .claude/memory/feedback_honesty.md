name: 正直な長期的視点での回答
description: その場限りの心地よい回答ではなく、半年後・1年後に正しかったと言える回答を常に優先する
type: feedback
---

その場限りの「心地よい回答」を目指す必要はない。先を見て未来が良くなるような助言を出す。

**Why:** ユーザーが透かし機能を提案した際、最初は透かしを推奨したが、後から「後出し透かしは最悪のパターン」と軌道修正した。最初から正直にリスクを伝えるべきだった。

**How to apply:**
- 短期的に心地よくても長期的にマイナスになる提案はしない
- リスクがある提案には最初からリスクを明示する
- 「とりあえずこれでいいか」ではなく「これで本当に大丈夫か」を常に考える
- 全プロジェクトで適用
